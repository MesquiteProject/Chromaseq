Q: Phred/phrap/chromaseq failed to assemble the different reads of a sequence into a contig automatically. Is there a way to assemble the contig by hand?


A: Yes, though it's still primitive. See the page on Multiple Reads/Contigs at Once.

 

Q: I have used Chromaseq and Phred and Phrap to produce a sequence. I've also made more base calls using Chromaseq's chromatogram viewer. However, one part of that sequence isn't the best, and I have resequenced part of the gene. Can I add these new chromatograms to the existing contig?

A: Not yet. You will need to reprocess the original chromatograms again for that sequence, and remake any base calls. We will add the ability to add new reads in the future.

 

Q: If I open up a chromatogram viewer window, and change the settings on it (for example, to show the primer sites), and then close the window and open another one, the settings always revert back to where they were - those changes I just made were forgotten. What can I do so that it remembers the settings?

A: The way settings in the chromatogram viewer are remembered is a bit unusual. To make sure that they are remembered, you need to set up the windows the way you want, and choose Chromatograms>Save Current Settings as Defaults.

 

Q: Can I insert bases into a sequence?

A: Yes, using the Insert Gap tool.

 

Q: I would like to use Chromaseq to make my base calls for a matrix of sequences, then align the matrix. Can I do this?

A: Yes, but only if you do the alignment from within Mesquite so that it is fully aware of the re-alignment at it happens. For example, you can use the tools in the Matrix>Align Multiple Sequences submenu.

 

Q: Can I reverse complement the sequences in the editor and still have Chromaseq function to let me look at the contigs and chromatograms?

A: Yes, if you use the Matrix>Alter/Transform>Reverse Complement to reverse and complement the entire sequence. If you reverse complement parts of the sequence, or use a tool outside of Mesquite, the connections between the bases in the editor and the bases in the chromatograms will be lost.

 

Q: Does Chromaseq do everything that a commercial program like Sequencher™ does?

A: Chromaseq is not intended as a full-featured replacement to programs like Sequencher™ or Aligner™. Chromaseq was built by us for our personal needs in systematics. We obtain many sequences of a few genes from beetles and spiders; many of these sequences are high-quality, and don't require much processing other than assembly of the read, trimming, and changing a few base calls. However, the tedium of processing these many sequences, and the lack of integration of existing tools within a systematics framework, led us to develop a solution that worked for us. While it has many custom features for dealing with the sorts of sequence data one acquires typically in systematics, it does not have the some features one would want in a full chromatogram analysis package. For example, you can't currently ask to simply recontig two reads together using a different stringency as you can with full chromatogram analysis packages.
